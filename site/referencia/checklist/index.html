    📖 COMPLETAR DOCUMENTACIÓN DEL SITIO¶

    🎯 Esto es lo que necesitas hacer¶

    El sitio de documentación ha sido configurado correctamente. Ahora necesitas completar los archivos de contenido que faltan.

    📁 Archivos que Necesitas Crear/Editar¶

    En docs/manual-tecnico/¶

    Copia estos archivos desde el directorio raíz al sitio de documentación:

    # De la raíz del proyecto
cp MANUAL_TECNICO_COMPLETO.md docs/manual-tecnico/01-estructura.md
cp DIAGRAMAS_TECNICOS.md docs/diagramas/
cp GUIA_RAPIDA_REFERENCIA.md docs/guia-rapida/referencia.md

    O manualmente:

    1. 01-estructura.md - Contenido de MANUAL_TECNICO_COMPLETO.md (Sección 1-2)
    2. 02-apis.md - Contenido de MANUAL_TECNICO_COMPLETO.md (Sección 2)

    🚀 Instalar y Ejecutar¶

    Paso 1: Instalar Dependencias¶

    cd c:\Users\ASUS\OneDrive\Documentos\Proyect

python setup_docs.py install

    Paso 2: Servir Localmente¶

    python setup_docs.py serve

    Abre: http://localhost:8000

    Paso 3: Generar Sitio Estático¶

    python setup_docs.py build

    Sitio en: site/ (listo para producción)

    Paso 4: Generar PDFs¶

    python setup_docs.py pdf

    PDFs en: site/pdf/

    📝 Personalizar el Sitio¶

    Edita mkdocs.yml para:

    # Cambiar nombre
site_name: Mi Documentación

# Cambiar tema
theme:
  name: material
  logo: assets/tu-logo.png
  favicon: assets/favicon.ico
  palette:
    primary: teal
    accent: orange

# Agregar analítica
extra:
  analytics:
    provider: google
    property: GA-XXXXX

    📤 Publicar en Línea¶

    GitHub Pages (Gratis)¶

    # 1. Estár en rama main
git checkout main

# 2. Deploy automático
python setup_docs.py deploy

# El sitio estará en:
# https://username.github.io/drivemarket

    Netlify (Gratis)¶

    1. Ve a https://netlify.com
    2. Conecta tu repo GitHub
    3. Configura:
    4. Build: mkdocs build
    5. Output: site/
    6. Deploy automático en push

    Servidor Propio¶

    # 1. Build
mkdocs build

# 2. Subir carpeta 'site/' al servidor
scp -r site/ user@servidor:/var/www/docs

# 3. Configurar en nginx
server {
    listen 80;
    server_name docs.miempresa.com;
    root /var/www/docs/site;
}

    🎨 Personalizar Estilos¶

    Crea docs/stylesheets/extra.css:

    /* Tus estilos personalizados */
:root {
  --md-primary-fg-color: #2c3e50;
  --md-accent-fg-color: #3498db;
}

body {
  font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
}

    📊 Agregar Google Analytics¶

    En mkdocs.yml:

    extra:
  analytics:
    provider: google
    property: G-XXXXXXXXXX

    🐛 Troubleshooting¶

    Comando no encontrado¶

    # Verifica que mkdocs esté instalado
pip list | grep mkdocs

# Reinstala si es necesario
python setup_docs.py install

    Cambios no se ven¶

    # Limpia caché
python setup_docs.py clean

# Reconstruye
mkdocs build

# Recarga navegador (Ctrl+Shift+R)

    Puerto 8000 en uso¶

    # Cambia puerto
mkdocs serve -a 127.0.0.1:9000

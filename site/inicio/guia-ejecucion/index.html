    ⚡ Guía de ejecución rápida - 3 pasos¶

    Paso 1️⃣: Instalar dependencias¶

    Abre una terminal en la carpeta del proyecto y ejecuta:

    python setup_docs.py install

    O si prefieres manual:

    pip install mkdocs mkdocs-material mkdocs-pdf-export-plugin pymdown-extensions mkdocs-awesome-pages-plugin

    ⏱️ Tiempo: ~2-3 minutos

    Paso 2️⃣: Ver el sitio en local¶

    Ejecuta:

    python setup_docs.py serve

    Luego abre tu navegador en:

    👉 http://localhost:8000

    Deberías ver un sitio profesional con: - Sidebar con navegación - Búsqueda integrada - Modo oscuro/claro - Documentación completa

    ⏱️ Tiempo: Inmediato

    Paso 3️⃣: Compilar para producción¶

    Para generar el sitio estático (HTML + PDF):

    python setup_docs.py build

    Esto crea la carpeta site/ con: - Archivos HTML listos para desplegar - Archivos PDF para descargar - Todos los assets necesarios

    ⏱️ Tiempo: ~30 segundos

    🚀 Desplegar a producción (Opcional)¶

    GitHub Pages (más fácil)¶

    # 1. Asegúrate de tener git configurado
git config user.name "Tu nombre"
git config user.email "tu@email.com"

# 2. Sube la documentación a GitHub Pages
python setup_docs.py deploy

    Sitio disponible en: https://username.github.io/drivemarket/

    Otro servidor¶

    # 1. Compilar
python setup_docs.py build

# 2. Copiar contenido de 'site/' a tu servidor
# Ejemplo con scp:
scp -r site/* usuario@servidor:/var/www/docs/

    ❓ Problemas comunes¶

    Error: "ModuleNotFoundError: No module named 'mkdocs'"¶

    Solución: Instala las dependencias primero

    python setup_docs.py install

    El sitio no se ve bien¶

    Solución: Limpia la caché de compilación

    rm -r site/
python setup_docs.py build

    PDF no se genera¶

    Solución: Requiere Chrome instalado

    # Instala Chrome si no lo tienes
# Windows: https://www.google.com/chrome/
# Mac: brew install --cask google-chrome
# Linux: sudo apt install google-chrome-stable

    Puerto 8000 ya está en uso¶

    Solución: Usa otro puerto

    mkdocs serve --dev-addr 127.0.0.1:8001

    📚 Archivos principales¶

    Archivo Propósito
    mkdocs.yml Configuración del sitio
    setup_docs.py Script de automatización
    requirements-docs.txt Dependencias
    docs/ Contenido de documentación
    docs/stylesheets/extra.css Estilos personalizados
    site/ Sitio compilado (se genera)

    🎯 Próximos pasos¶

    1. ✅ Instalar → python setup_docs.py install
    2. ✅ Probar → python setup_docs.py serve
    3. ✅ Compilar → python setup_docs.py build
    4. 🔄 Desplegar → python setup_docs.py deploy
    5. 📝 Mantener → Actualizar archivos en docs/

    💡 Tips¶

    • Editar contenido: Modifica archivos en docs/ y recarga el navegador
    • Agregar sección: Crea un .md nuevo en docs/ con estructura
    • Cambiar estilos: Edita docs/stylesheets/extra.css
    • Actualizar mkdocs.yml: Si cambias la config, reinicia el servidor
